# Plan 02-03: Home Page Assembly

## Objective

`app/page.tsx`를 Phase 2 요구사항을 충족하는 서버 컴포넌트로 재구현한다.

- `searchParams.category`로 카테고리 필터링 (서버사이드 Prisma where 절)
- 카테고리 목록을 DB에서 동적으로 읽어 CategoryFilter에 전달
- JobCard 그리드: 1열(모바일) / 2열(태블릿) / 3열(데스크탑)
- 직업 0개인 경우 빈 상태 UI 표시

**중요:** 카테고리 수를 절대 하드코딩하지 않는다. 현재 DB는 2개 카테고리지만
향후 변경될 수 있으므로 항상 DB에서 동적으로 로드한다.

  <action>
    app/page.tsx를 아래 스펙으로 전체 재작성한다.

    컴포넌트 구조:
    - Server Component (async 함수)
    - props: { searchParams: Promise<{ category?: string }> }
      (Next.js 15+ — searchParams는 Promise)

    Prisma 쿼리:
    1. categories: prisma.category.findMany({ orderBy: { sortOrder: "asc" } })
    2. jobs: prisma.job.findMany({
         where: {
           isPublished: true,
           ...(activeCategory ? { category: { slug: activeCategory } } : {}),
         },
         include: { category: true },
         orderBy: [{ category: { sortOrder: "asc" } }, { title: "asc" }],
       })
    두 쿼리를 Promise.all로 병렬 실행.

    activeCategory: `(await searchParams).category`

    레이아웃 구조:
      <main className="min-h-screen px-5 py-8 sm:px-8 lg:px-12">
        <section className="mx-auto flex max-w-6xl flex-col gap-8">
          {/* 헤더 — 기존 Phase 1 헤더 유지 또는 간소화 */}
          <header className="flex flex-col gap-3 border-b border-[var(--line)] pb-6">
            <p className="text-sm font-semibold text-[#246b57]">CS Career Compass</p>
            <h1 className="text-3xl font-bold sm:text-4xl">IT 진로 카탈로그</h1>
            <p className="text-sm text-[var(--muted)]">
              컴퓨터공학부 학생을 위한 국내 IT 직군 안내
            </p>
          </header>

          {/* 카테고리 필터 */}
          <CategoryFilter categories={categories} activeSlug={activeCategory} />

          {/* 직업 그리드 */}
          {jobs.length > 0 ? (
            <div className="grid gap-4 sm:grid-cols-2 lg:grid-cols-3">
              {jobs.map((job) => <JobCard key={job.id} job={job} />)}
            </div>
          ) : (
            {/* 빈 상태 UI */}
            <div className="flex flex-col items-center gap-3 py-16 text-center">
              <p className="text-[var(--muted)]">해당 카테고리에 직업이 없습니다.</p>
              <Link href="/" className="text-sm text-[#246b57] underline">전체 보기</Link>
            </div>
          )}
        </section>
      </main>

    import 목록:
      import Link from "next/link"
      import { prisma } from "@/lib/db"
      import { CategoryFilter } from "@/app/components/CategoryFilter"
      import { JobCard } from "@/app/components/JobCard"
  </action>
  <acceptance_criteria>
    - app/page.tsx에 `searchParams: Promise<{ category?: string }>` props가 존재한다
    - `const { category: activeCategory } = await searchParams;` 또는 동등한 패턴이 존재한다
    - Prisma category 쿼리가 `orderBy: { sortOrder: "asc" }` 를 사용한다
    - Prisma job 쿼리가 `where: { isPublished: true }` 를 포함한다
    - Prisma job 쿼리가 `include: { category: true }` 를 포함한다
    - activeCategory가 있을 때 `category: { slug: activeCategory }` 를 where에 추가한다
    - 그리드 className에 `sm:grid-cols-2` 와 `lg:grid-cols-3` 이 존재한다
    - CategoryFilter에 categories와 activeSlug 두 props가 모두 전달된다
    - jobs.length === 0일 때 빈 상태 UI 코드 경로가 존재한다
    - `npx tsc --noEmit` 실행 시 page.tsx에 대한 TypeScript 오류가 없다
  </acceptance_criteria>
</task>
