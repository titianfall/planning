# Plan 02-01: Schema Extension — exampleCompanies

## Objective

직업 카드(D-01)에 "예시 고용 회사명"을 표시하기 위해 Prisma `Job` 모델에 `exampleCompanies` 필드를 추가하고 마이그레이션을 실행한다. 기존 `skills` 필드와 동일하게 JSON string array로 저장한다.

## Tasks

<task id="T01" type="execute">
  <title>prisma/schema.prisma에 exampleCompanies 필드 추가 및 마이그레이션 실행</title>
  <read_first>
    - prisma/schema.prisma (현재 Job 모델 전체 확인)
    - prisma/migrations/20260516144657_init_local_foundation/migration.sql (기존 마이그레이션 패턴 확인)
    - .env (DATABASE_URL 확인)
  </read_first>
  <action>
    prisma/schema.prisma의 Job 모델에 다음 필드를 추가한다.
    위치: updatedAt 필드 바로 아래, category 관계 선언 위.

      exampleCompanies String @default("[]")

    추가 후 마이그레이션을 실행한다:
      npx prisma migrate dev --name add_example_companies

    마이그레이션 완료 후 Prisma Client를 재생성한다:
      npx prisma generate

    주의: DATABASE_URL이 "file:./dev.db" 인지 확인 후 실행.
    Docker 환경에서 작업 중인 경우: docker compose exec app npx prisma migrate dev --name add_example_companies
  </action>
  <acceptance_criteria>
    - prisma/schema.prisma Job 모델에 `exampleCompanies String @default("[]")` 라인이 존재한다
    - prisma/migrations/ 아래에 *add_example_companies* 이름을 포함한 새 디렉토리가 생성된다
    - 해당 migration.sql에 `ALTER TABLE "Job" ADD COLUMN "exampleCompanies" TEXT NOT NULL DEFAULT '[]'` 또는 동등한 구문이 포함된다
    - `npx prisma generate` 또는 migrate dev 이후 오류가 없다
  </acceptance_criteria>
</task>

<task id="T02" type="execute">
  <title>prisma/seed.ts의 각 직업 데이터에 exampleCompanies 추가</title>
  <read_first>
    - prisma/seed.ts (전체 파일 — 특히 jobs 배열 구조와 prisma.job.create data 블록)
    - prisma/schema.prisma (exampleCompanies 필드 타입 확인)
  </read_first>
  <action>
    prisma/seed.ts의 jobs 배열 각 항목에 exampleCompanies 필드를 추가한다.
    저장 방식: JSON string array (기존 skills 필드와 동일하게 `json()` 헬퍼 사용).

    각 직업의 exampleCompanies 값:
    - slug "backend-engineer":         ["네이버", "카카오", "쿠팡", "토스"]
    - slug "frontend-engineer":        ["카카오", "네이버", "크래프톤", "라인플러스"]
    - slug "devops-engineer":          ["네이버클라우드", "카카오", "NHN클라우드", "삼성SDS"]
    - slug "data-analyst":             ["카카오", "네이버", "당근마켓", "쏘카"]
    - slug "machine-learning-engineer":["카카오브레인", "네이버 클로바", "삼성리서치", "LG AI연구원"]
    - slug "data-engineer":            ["네이버", "카카오", "SK텔레콤", "현대자동차"]

    prisma.job.create의 data 블록에 다음을 추가한다:
      exampleCompanies: json(job.exampleCompanies),

    위치: skills: json(job.skills) 바로 아래.

    jobs 배열 타입 선언(있다면)도 exampleCompanies: string[] 를 포함하도록 수정한다.
  </action>
  <acceptance_criteria>
    - prisma/seed.ts의 jobs 배열 6개 항목 모두에 exampleCompanies 키가 존재한다
    - prisma.job.create data 블록에 `exampleCompanies: json(job.exampleCompanies)` 가 포함된다
    - `npx prisma db seed` 또는 `npm run db:seed` 실행이 오류 없이 완료된다
    - seed 실행 후 `npx prisma studio` 또는 DB 직접 조회 시 각 job의 exampleCompanies가 빈 배열([])이 아닌 배열 JSON 문자열을 포함한다
  </acceptance_criteria>
</task>
